## LIÇÕES APRENDIDAS

### 1. PROCESSAMENTO PDF
**Principais Desafios:**
- **Múltiplos Formatos:** Cada plataforma (Booking, Airbnb) tem layout diferente
- **Qualidade OCR:** PDFs escaneados têm precisão menor
- **Detecção Manuscritos:** Necessário serviço especializado (RolmOCR)
- **Fallback Automático:** Sistema deve funcionar mesmo com APIs em baixo

**Soluções Implementadas:**
- Pipeline multi-estágio com detecção automática de formato
- Sistema de confiança por percentagem (confidence score)
- Validação cruzada de dados extraídos
- Cache de resultados para evitar reprocessamento

### 2. SISTEMA DE ORÇAMENTOS
**Aprendizados:**
- **Preços Dinâmicos:** Sistema deve calcular automaticamente baseado em área/tipo
- **Templates PDF:** Necessário design profissional e campos variáveis
- **Aprovação Workflow:** Status tracking (pendente → aprovado → faturado)
- **Validade:** Orçamentos têm data de expiração automática

**Implementação:**
- Calculadora automática com base em m² e divisões
- Geração PDF com logo e dados empresa
- Sistema de versioning para alterações
- Notificações automáticas de expiração

### 3. LIGAÇÃO IA COM BASE DE DADOS
**Desafios Técnicos:**
- **Contexto Limitado:** IA não conhece estrutura específica dos dados
- **Queries Complexas:** Perguntas naturais → SQL complexo
- **Performance:** Consultas grandes podem ser lentas
- **Precisão:** Respostas devem ser factuais, não inventadas

**Arquitetura RAG Implementada:**
```
Consulta Usuário → Embedding → Busca Vetorial → Contexto Relevante → IA → Resposta
```

**Melhorias Aplicadas:**
- **Embeddings:** Vetorização de todos os dados para busca semântica
- **PostgreSQL + pgVector:** Busca vetorial eficiente
- **Prompt Engineering:** Instruções específicas para evitar alucinações
- **Validation Layer:** Verificação de respostas contra dados reais

### 4. SCHEDULER E AUTOMAÇÃO
**Problemas Iniciais:**
- **Sobreposição:** Múltiplas execuções simultâneas
- **Falhas Silenciosas:** Erros não reportados
- **Performance:** Impacto na base de dados

**Soluções:**
- Flag de execução para evitar overlap
- Logging detalhado de todas as operações
- Execução assíncrona com timeout
- Rate limiting nas chamadas API

### 5. GESTÃO DE ESTADO E CACHE
**Estratégias:**
- **Cache em Memória:** Para dados frequentemente acessados
- **Invalidação Inteligente:** Update automático quando dados mudam
- **Session Management:** PostgreSQL store para sessões
- **Connection Pooling:** Otimização conexões base de dados

## ESTRUTURA DE DADOS OTIMIZADA PARA IA

### Schema com Vetorização
```sql
-- Tabela principal para consultas IA
CREATE VIEW ai_queryable_data AS
SELECT 
  r.id as reservation_id,
  r.guest_name,
  r.check_in_date,
  r.check_out_date,
  r.total_amount,
  r.status,
  p.name as property_name,
  p.cleaning_cost,
  p.check_in_fee,
  o.name as owner_name,
  ct.name as cleaning_team,
  EXTRACT(MONTH FROM r.check_in_date::date) as month,
  EXTRACT(YEAR FROM r.check_in_date::date) as year
FROM reservations r
JOIN properties p ON r.property_id = p.id
JOIN owners o ON p.owner_id = o.id
LEFT JOIN cleaning_teams ct ON p.cleaning_team_id = ct.id;

-- Índices para performance
CREATE INDEX idx_ai_dates ON reservations(check_in_date, check_out_date);
CREATE INDEX idx_ai_amounts ON reservations(total_amount);
CREATE INDEX idx_ai_status ON reservations(status);
```
